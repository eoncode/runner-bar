What RunnerBar does

Monitor workflows, manage local runners, and auto-recover from failures — all from the menu bar.

01 — Actions

Your workflow runs, all in one place

Expand any workflow into a live tree: Workflow → Jobs → Steps. See elapsed time, status, and live progress at a glance.

Copy logs, cancel, or re-run directly from the context menu — no browser tab required.

a3f92c1 Update deploy config SUCCESS
└ Build & Test 01:24
└ Checkout 00:03
└ swift build 01:18
└ Run tests 00:41
02 — Local Runners

See and control every runner on your Mac

Provision new local runners in one click. The app handles the token, the install command, and the registration. See live status of all your runners.

Toggle them on and off and deregister and remove them when they are not needed anymore, in one click. You can also add pre-existing local runners on your mac.

03 — AI Failure Hooks

Auto-recover failures with any AI CLI

Set a failure hook per repo or branch. When a run fails, the app fires your custom shell command in Terminal — log, metadata, branch, and run link injected automatically.

Let agents like Claude Code, Codex, Gemini CLI, or Aider diagnose the failure, push a fix, or open a PR — so it's waiting for you in the morning.

Built for the agentic era

Self-hosted runners change the economics of CI — especially in the Agentic coding era. Cloud CI made sense before AI started committing code every 30 seconds. But in the AI era, cloud CI is too slow and too expensive to handle active AI development.

🚀

Up to 5x faster builds

Self-hosted macOS runners are up to 5× faster than GitHub-hosted ones — persist caches, zero queue time, no throttling and as much parallelism your hardware can handle. Your builds always have priority.

💸

Cut your Actions bill

Cloud macOS minutes are expensive, and Agentic coding workflows burn through them fast. Active coding can cost as much as $200–300+ per developer per day in the cloud if you run a full CI suite and commit 100s of times a day. Running locally costs nothing.

🤖

Fix failures automatically

Think reacting to a dependency attack the second it hits. Or when there is a failing micro service somewhere. Run UI-tests around the clock and get notified the second something breaks. This lets you stay on top of things even when you sleep.

Get started in minutes

1

Install with one command

Paste the curl command in Terminal. RunnerBar installs and launches automatically.

2

Sign in with GitHub

Automatically uses your CLI github_token. Optionally sign in with one click via OAuth.

3

Add a repo or org scope

Enter an owner/repo slug or an org name. Monitor a single repo or an entire GitHub org.

4

Add & manage local runners (Optional)

Provision new or add pre-existing runners. start, stop, or remove them directly from the UI.

5

Configure a failure hook (optional)

Set a shell command in Settings to fire your AI CLI automatically whenever a run fails.
