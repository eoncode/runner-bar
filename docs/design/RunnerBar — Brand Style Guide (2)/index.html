Brand Identity

Who RunnerBar is, in one breath.

Positioning

A macOS-native menu bar utility that puts GitHub Actions, self-hosted runners, and AI failure recovery one click from the system tray.

Category

Personality

01

Workflow status

Live runs across all repos and orgs — drilled into jobs, steps, and logs from a single popover.

02

Local runner manager

Auto-discovers runners on this Mac. Start, stop, add, remove — without touching Terminal or github.com.

03

Failure hooks

When a run fails, fire a shell command with $FAILURE_LOG, $RUN_LINK, and friends. Default: hand the log to Gemini.

Logo

A single bolt. Energy. A run firing. The hook landing. One mark, four contexts.

RunnerBar
Horizontal lockup

Clear space

Minimum padding around the bolt equals the width of its narrowest leg (≈ 1/8 of the icon canvas) on every side.

Minimum size

Outline mark: 16 px (12 px for the status bar template). Lockup: 96 px wide — below that, drop the wordmark.

Wordmark

Always set in Satoshi 700 with capital R and B. One word, no space, no slash. The bolt sits left of the wordmark, never above.

Don't

Don't rotate, mirror, recolor with multiple hues, fill the bolt, stretch, or place on busy photography. Outline only — always.

Rule of one accent

RunnerBar uses Action Blue as its single brand accent. Status colors are functional, not decorative — they appear only where the system reports actual state. Avoid gradients across multiple hues.

Type rules
  • Maximum three families: Satoshi (display), General Sans (UI & body), Geist Mono (mono).
  • Display sizes only above 24 px. Body sizes never above 20 px.
  • Mono is reserved for code, paths, shell variables, run IDs, hotkeys, and SHAs. Never set marketing prose in mono.
  • Tracking tightens as size grows. Body copy stays at default tracking.

Icon System

One bolt. Three platforms. Five status states. The mark is the message: a CI run firing.

The metaphor

A bolt is the smallest unit of RunnerBar: a workflow firing, a hook landing, a runner waking up. It carries the brand at 12 px in the menu bar and at 1024 px on the App Store.

macOS app icon · 1024 × 1024

macOS · default
macos-icon.svg → 1024.png

Recipe

  • Continuous-corner squircle · 228 px radius on a 1024 canvas (~22%).
  • Background: radial blue-to-ink gradient anchored top-left.
  • Faint 64 px blueprint grid for technical density.
  • Bolt: outline only — stroked with an Action Blue gradient (#7CC0FF → #0066D6), 1.4 stroke at 24px source, round joins.
  • Glassy top sheen (Tahoe-era) over everything.
  • Export sizes: 16 32 64 128 256 512 1024 for AppIcon.appiconset.

iOS app icon · light · dark · tinted

iOS 18+ export rules
  • All three variants are 1024 × 1024 PNG, fully opaque, no pre-rounded corners — the system applies the squircle mask.
  • Tinted variant uses grayscale only. Depth comes from stop-opacity on the bolt gradient (100% → 55%).
  • Add all three to the asset catalog under one AppIcon entry with Appearances: Any, Dark, Tinted.

macOS status bar icon · 22 pt template

Recipe

  • 22 pt canvas, exported @1x/@2x/@3x as 22/44/66 px PNGs.
  • Stroked outline at 1.6 pt with rounded joins. Bolt is scaled to 0.78× of source for a 2 pt optical margin.
  • Marked template image in Swift: image.isTemplate = true. macOS handles light/dark/accent tinting automatically — do not bake in color.
  • Running state: same outline tinted Action Blue with a soft glow. Success / failure: outline plus a green or red dot. State variants override the template flag.

Swift

// Status bar item with template image
let statusItem = NSStatusBar.system.statusItem(withLength: NSStatusItem.variableLength)
if let button = statusItem.button {
  let image = NSImage(named: "StatusBolt")
  image?.isTemplate = true
  button.image = image
}

Where each icon lives

Status bar (22 pt)

The only icon the user sees daily. Template image, monochrome, outline by default — fills in when a run is active.

Dock & Finder (macOS)

Full gradient icon. Appears in Spotlight, Dock, Finder, and the About panel.

Home Screen (iOS)

Three variants ship together. The system swaps based on user appearance preference.

README & web

Use the macOS icon at 96–128 px for the README hero. Use the outline mark for inline copy and favicons.

Hero structure

Eyebrow chip → tight headline (max 7 words) → one-sentence subhead → install command as the primary CTA → secondary "GitHub" link.

Imagery

The Liquid Glass popover, floating over a desaturated macOS desktop. No stock photography. No 3D renders.

Backdrop

Ink background with a faint blueprint grid. Action Blue glow may bloom from behind the headline — never as a gradient ramp.

Wordmark in marketing

Use "RunnerBar" as the headline anchor only once. In body copy, treat it as a regular brand name without bold or accent color.
